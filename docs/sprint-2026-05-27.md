Coverage spread:

- **CSS** (57): text/inline, logical properties, sizing keywords, selectors, pseudo-elements, @-rules, visual effects, scrolling/containment, forms, animation
- **DOM/JS** (21): querySelector + addEventListener + classList + style setters foundation, plus alert/confirm, console, location/history/title, navigator, encoding APIs, FormData, DOMParser, fetch, MutationObserver, rAF, URL/URLSearchParams
- **Rendering** (29): SVG opacity, ::marker, Canvas 2D, <details>/<summary>, <picture>/srcset, alt-text fallback, iframe support, image-rendering, etc.
- **Chrome** (11): Find-in-page, bookmarks, history UI, view source, save page, print, right-click menu, about: diagnostic pages, tabs
- **Networking** (9): cookies/localStorage, Brotli, form POST, macTLS ALPN + TLS 1.3, plus the won't-fix docs (HTTP/2, WebSockets, etc.)
- **Build** (2): Mac duplicate-source-tree workflow, Remove Object Code automation

Hard structural items (#37 filter, #38 caption-side, #39 table-layout: fixed, #66 subgrid, #80 appearance, #109 Canvas 2D, #91 gui_layout_table refactor) are explicitly marked priority-low / deferred / V2 so contributors aren't lured into the trap zones.

The **27 good-first-issue** tagged items are deliberately scoped to bounded surfaces — pure cssh_css preprocessor passes, single-property paint-site changes, parser audits, isolated JS globals — so anyone curious about contributing to a Mac OS 9 browser can pick one up without spelunking the whole codebase.

Four issues are tagged **wontfix** with documentation (#127 WebSockets/EventSource, #128 Web Workers/Service Workers, #129 HTTP/2/HTTP/3, #130 WebGL/WebRTC/Crypto/Geolocation) so contributors don't waste effort on architecturally-out-of-reach APIs.
