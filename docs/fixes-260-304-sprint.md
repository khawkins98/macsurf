## Where we ended

- Build links cleanly.
- App launches into a Carbon window with chrome (URL bar, Back/Fwd/Reload/Home, scroll bars, status bar).
- All NetSurf core init runs to completion. `main()` reaches the event loop.
- **Last open issue:** pages submitted via URL bar / Home navigate but never appear to load. The fetcher state machine was inert because `macos9_schedule_run()` was never wired into the event loop — fixes304 wires it. As of writing, that's the most recent fix and untested by the user.

## Where we started

The sprint started after the v0.3 rendering work — fixes260 onward. The build environment had drifted: many files in `MacSurf.mcp` were stale stubs (`dt_*.c`, `*_stub.c`), the porting work for libcss/libdom/libhubbub/libparserutils was incomplete, and CW8 was hitting cascading C89 / Carbon-header errors. The April-22 working binary (`builds/MacSurf.hqx`) was the last known-good state.

## High-level phases of the sprint

### Phase 1 — CW8 C89 + Carbon.h chain (fixes260-269)

**Pattern:** Carbon.h sub-headers internally use types defined in headers that hadn't been processed yet. Each suppression unmasked the next one.

- **fixes260** — NSLOG `__VA_ARGS__` macro. CW8 C89 rejects fixed-param macros called with extra args. Switched `NSLOG(cat, level, ...)` to use varargs in `macsurf_prefix.h`. Made `html/html.h` and `html/form_internal.h` shims forwarders rather than guard-stealing replacements.
- **fixes261** — Removed dead `#include "utils/list.h"` from `libdom/src/events/event.h`. CW8 cannot resolve relative includes from headers found via access path.
- **fixes262** — Multiple C89 fixes: `css__parse_calc` int32_t signature, `autogenerated_computed.h` "calc.h" (no path prefix), `cookie_manager.c` time.h, `netsurf/layout.h`+`window.h` need `utils/errors.h` for `nserror`.
- **fixes263-265** — Carbon.h chain: pre-define `__INTERNETCONFIG__` to skip InternetConfig.h (uses `AliasRecord` by value), pre-declare `AliasHandle` for MacWindows.h, suppress `__KEYCHAINCORE__`. Multiple C89 fixes including `(css_fixed_or_calc)0` GNU union cast → C89 compound-statement local var.
- **fixes266** — Carbon.h chain continued: suppress `__ALIASES__` (with AliasHandle forward decls), `__KEYCHAINCORE__`, `__ATSLayoutTypes__`. `page-info.c` full C89 conversion (designated initializers, hoisted enum, runtime-init for pi__entries). `ns_time.c` removed broken `_TIME_H` workaround.
- **fixes267-268** — `utils/string.h` was being picked up via access path as `<string.h>` (shadowing MSL). Eventually inlined the `nserror` enum directly in `utils/string.h` so it has no external #include dependency.
- **fixes269** — Suppress `__KEYCHAINHI__` via Carbon.h:210 chain.

### Phase 2 — libcss `_ALIGNED` cascade (fixes271-277)

**Pattern:** `_ALIGNED` was used as a struct-close tag in `libcss/src/stylesheet.h` but never defined as a macro. CW8 read it as a global variable declaration → every TU including stylesheet.h emitted a duplicate `_ALIGNED` symbol → multi-link conflict across hundreds of files.

- **fixes271** — Dropped `} _ALIGNED;` from `libcss/src/stylesheet.h` → `};`.
- **fixes272-277** — Defensively `#define _ALIGNED` empty in: `macsurf_prefix.h`, `stylesheet.h`, `propstrings.h`, `select.h`, `bytecode.h`, `parse/properties/utils.h`, `select/properties/properties.h`, `computed.h`, `language.h`, `mq.h`, `arena.h`, `dispatch.h`, `calc.h`, `lex.h`, `font_face.h`, `important.h`, `propget.h`, `propset.h`, `unit.h`, plus the 3 stubborn .c files (`computed.c`, `flex.c`, `p_utils.c`) where the include chain didn't reach the prefix in time.

### Phase 3 — Stub overlap pruning (fixes278-286)

**Pattern:** As real ported `.c` files were added to MacSurf.mcp, `*_stub.c` placeholders started conflicting at link time.

- **fixes278** — Implemented `macos9_handle_mouse_down` / `macos9_handle_key_down` (FindWindow dispatch + scroll-bar / button click routing + URL-field TEKey + arrow/PgUp/PgDn/Home/End scrolling). Neutered conflict stubs in `misc_stub.c`. Added `nsu_base64_encode_url` stub.
- **fixes281** — Stripped `misc_stub.c` to remove `fetch_abort`, `search_web_*`, `dom_namespace_finalise`, `lwc_iterate_strings`, `save_pdf`. All have real implementations now.
- **fixes282** — Added `N_ELEMENTS`/`UNUSED` macros globally in `macsurf_prefix.h` (libcss/libhubbub/libparserutils source files include `utils/utils.h` but the access path resolved to NetSurf's utils.h, which lacks them). Added libcss helpers (`isDigit`/`isHex`/`charToHex`/`css_error_from_*`/`css__number_from_lwc_string`), `nsu_getmonotonic_ms`, `fetch_poll`/`netsurf_poll`/Console stubs.
- **fixes283** — POSIX shims (`strdup`/`strcasecmp`/`strncasecmp`/`strtof`/`mkdir`/`stat`) and non-static `get_border_radius`/`get_box_shadow`/`get_macsurf_gradient` stubs. *(Most of these were later removed in fixes298 — see Phase 6.)*
- **fixes284** — `stat` signature → `struct stat*` to match `shims/stat.h`.
- **fixes285** — Dropped duplicate `css__number_from_lwc_string` (provided by `libcss/src/utils/css_utils.c`; `lwc_string_data`/`_length` are macros, not functions).
- **fixes286** — `plotters.c` removed `static` qualifier from `macos9_font_id_from_style` / `macos9_face_from_style` (called across TUs from `macos9_font.c`).

The user removed many obsolete `.c` files from MacSurf.mcp during this phase per the per-fix instructions: `dt_bitmap.c`, `dt_hotlist.c`, `dt_print.c`, `macos9_http_fetcher_crlf.c`, `corestrings_stub.c`, `misc_stub.c` (later re-added with stripped contents), `ns_content.c`, `http_stub.c`, `dt_selection.c`, `dt_global_history.c`, `macos9_fetcher_stubs_crlf.c`, `browser_history_stub.c`, `lwc_stub.c`, `dt_local_history.c`, `dt_download.c`, `dt_search.c`, `macos9_fetcher_init.c`.

### Phase 4 — Init wiring (fixes287-291)

**Pattern:** With binary linking, app started crashing at startup (`__ptr_glue` stack ending at `llcache_initialise`).

- **fixes287** — `main.c` wired `macos9_table.llcache = null_llcache_table`. Without it `guit->llcache->initialise(...)` walked off NULL.
- **fixes288** — Wired `macos9_table.fetch = &macos9_fetch_table`. `netsurf_register` was failing silently because `verify_fetch_register(NULL)` returned `BAD_PARAMETER`. The previous `__ptr_glue` crash was actually a NULL function-pointer call inside an llcache_initialise that never had the table.
- **fixes289-291** — Open Transport wiring. Tried `InitOpenTransport()` (link failed; not exported by this CarbonLib), reverted to `InitOpenTransportInContext` with manual `#define kInitOTForApplicationMask 0x00000002`.

### Phase 5 — Diagnostic / "main never runs" diagnosis (fixes294-296)

**Pattern:** After all init wiring, the debugger session would close immediately on hitting Play. No beep from `SysBeep(20)` at top of main meant CFM rejected binary or the runtime crashed before main.

- **fixes294** — `SysBeep` instrumentation at very top of main. No beep heard → main not entered.
- **fixes295** — Stripped `main()` to bare minimum (SysBeep + event loop). Still no beep.
- **fixes296** — Reverted `main.c` to clean state (dropped diagnostic clutter, kept `llcache`/`fetch` wiring + `kInitOTForApplicationMask` define). `window.c` removed function-pointer cast on `macos9_gw_get_scroll` (return-type mismatch was masking a real bug).

### Phase 6 — Carbon resource + MSL shadowing (fixes297-299)

**This is where the real "main not entering" bug was found.**

- **fixes297** — `MacSurf.r` (Rez source) re-shipped instead of `MacSurf.rsrc` (binary). The CW8 File Mappings panel maps `.rsrc` extension to **Rez compiler**, which silently failed to parse the binary resource fork. Without `'carb'` resource the binary was treated as classic PEF, CarbonLib never loaded, and the first Carbon call crashed silently. **REMOVE `MacSurf.rsrc` from MacSurf.mcp; ADD `MacSurf.r` and `Types.r`.**
- **fixes298** — **The real `__start` blocker.** `misc_stub.c` was shadowing real MSL_All_Carbon.Lib implementations of `InstallConsole`, `RemoveConsole`, `WriteCharsToConsole`, `ReadCharsFromConsole`, `strdup`, `strcasecmp`, `strncasecmp`, `mkdir`, `stat`. Our stubs returned 0 from `InstallConsole` (claiming success) but didn't actually set up Carbon console internals. MSL's stdio init succeeded nominally but FILE* tables were uninitialized → first stdio touch crashed before `main()`. Stripped all those stubs from `misc_stub.c` (kept only `strtof` and `nsu_getmonotonic_ms`). Added `HAVE_UTSNAME`/`HAVE_REALPATH`/`HAVE_STRDUP`/`HAVE_STRCASECMP`/`HAVE_STRNCASECMP` to `macsurf_prefix.h` to keep `utils.c`'s in-tree stubs from compiling.
- **fixes299** — Reverted `HAVE_*` defines from `macsurf_prefix.h` (`config.h` already controls these conditionally; my defines collided). The Console stub removal in fixes298 was the load-bearing fix.

After fixes298, **`main()` finally runs**.

### Phase 7 — First-run init bugs (fixes300-301)

- **fixes300** — `window.c update_button_states` — guard `g->bw` NULL before `browser_window_*` calls. Assertion `bw != NULL` fired during initial window creation when `bw` was not yet attached.
- **fixes301** — `macos9.h` dropped stale `frogfind` `MACSURF_HOME_URL` define. Canonical value is in `macsurf_config.h` (`http://mac.mp.ls/simple.html`). `window.c` now includes `macsurf_config.h`.

### Phase 8 — Page-load failure (fixes302-304)

- **fixes302** — Re-enabled `macsurf_debug_log_init()` unconditionally (was gated behind `#ifdef MACSURF_DEBUG`). Added MS_LOG checkpoints in `main()`, `macos9_window_navigate`, `macos9_window_address_bar_submit`, `macos9_http_setup`, `mfs_open`.
- **fixes303** — CW8 C89 fix: moved `MS_LOG` calls after variable declarations in `mfs_open` / `macos9_http_setup`.
- **fixes304** — **Last shipped, not yet user-tested.** `macos9_poll` was never calling `macos9_schedule_run()`. NetSurf's `fetcher_poll` self-schedules via `guit->misc->schedule(...)` — the callback got queued but the queue was never drained. Pages submitted to fetch sat forever. Wiring `macos9_schedule_run()` into the per-tick event loop should let the fetch state machine progress.

## Known design decisions worth preserving

1. **Plain Open Transport vs. InContext** — CLAUDE.md "OT Rules" recommend plain. Reality on this CarbonLib build: `InitOpenTransport()` / `CloseOpenTransport()` are NOT exported. Use `*InContext` variants with manual `#define kInitOTForApplicationMask 0x00000002`.

2. **`'carb'` resource** — must come from a Rez source file (`MacSurf.r`, `Types.r`), not a binary `.rsrc` file. CW8's File Mappings maps `.rsrc` extension to the Rez compiler which fails silently on binary input.

3. **Don't shadow MSL** — never stub `InstallConsole`, `RemoveConsole`, `Read/WriteCharsToConsole`, `strdup`, `strcasecmp`, `strncasecmp`, `mkdir`, `stat` — `MSL_All_Carbon.Lib` provides real implementations and the C runtime startup depends on them.

4. **Scheduler must be driven** — `macos9_schedule_run()` MUST be called every event-loop tick. Per fixes304, this is now wired into `macos9_poll`. If it's removed for any reason, fetches and other scheduled callbacks freeze.

5. **`#define _ALIGNED` empty everywhere libcss might use it** — defensive measure; the original `} _ALIGNED;` in `stylesheet.h` was fixed in fixes271, but the empty-define in many headers prevents regressions if any auto-gen or upstream pull reintroduces it.

## Diagnostics in code

The `MS_LOG()` instrumentation and `macsurf_debug_log_init()` write to `MacSurf Debug.log` on the Desktop. Useful checkpoints currently in code:

- main: startup steps (`InitOT OK`, `Appearance OK`, `netsurf_register done`, `nsoption_init done`, `netsurf_init done`, `http_fetcher registered`, `initial window created`)
- window.c: `URL submit fired`, `navigate:`, `nav: no bw`, `nav: nsurl_create FAIL`, `nav: calling browser_window_navigate`, `nav: done`
- macos9_http_fetcher.c: `http_setup called`, `mfs_open called`

Strip these once stable, but don't strip the log infrastructure itself — the file-backed channel survives crashes that title-bar / `printf` instrumentation can't.
