MacSurf — fixes143a (list markers)

fixes143a replaces the U+2022 BULLET that rendered as a semicolon-looking glyph on G3 Helvetica TT with U+00B7 MIDDLE DOT, which the existing conversion table maps to MacRoman 0xE1 -- a different font slot and a clean small dot.

PROBE L1 — disc (default unordered)

  • Disc alpha
  • Disc beta
  • Disc gamma

EXPECT: a small centred dot before each item. Pre-fixes143a this rendered as ";" on G3 hardware.

PROBE L2 — circle

  • Circle alpha
  • Circle beta

EXPECT: ASCII "o" before each item. MacRoman has no hollow-circle glyph; the conversion table substitutes ASCII 'o' for U+25CB.

PROBE L3 — square

  • Square alpha
  • Square beta

EXPECT: same MacRoman bullet slot (0xA5) the old disc used. If this ALSO renders as ";" then 0xA5's glyph is the underlying problem and we'll switch square to an ASCII fallback in a follow-up round.

PROBE L4 — decimal (ordered)

  1. One
  2. Two
  3. Three

EXPECT: "1. 2. 3." -- numerical markers render via layout.c's counter formatter, not through the glyph-substitution path. Unchanged by fixes143a.
